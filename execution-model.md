# Execution Model

## What it is

This page explains how execution feedback and settlement are handled in SSS.

## Why it matters

In modern trading systems, speed and finality are not always the same thing. Users need to understand what happens immediately and what is finalized later.

## How it works

SSS distinguishes between:

- Fast feedback for user interaction
- On-chain finalized settlement for verifiable results

This model helps create a smoother trading experience while preserving on-chain trust.

## Summary

- Fast response improves UX
- Finalized settlement preserves verifiability
- The two should be understood separately
